# 칼로리 계산 정확도 전면 개선

- **ID**: 006
- **날짜**: 2026-04-14
- **유형**: 버그 수정

## 작업 요약
식약처 API 카테고리 매핑 누락(12개), servings_per_food 이중 스케일링, 모호 식품 분류 오류, 간식/점심 배식량 미분기 등 칼로리 계산 정확도 문제를 전면 수정. GPT 참조 식단 대비 평균 오차 2.4kcal로 개선.

## 변경 파일 목록

### nutrition_api.py
- `cat_map`에 누락된 12개 식약처 API 카테고리 추가 (죽 및 스프류, 유제품류 및 빙과류, 전·적 및 부침류, 수·조·어·육류, 떡류, 한과류, 샐러드류, 젓갈류, 장아찌류, 음료 및 차류, 두류)
- `BASIC_INGREDIENTS`에 단호박, 찐단호박, 고구마, 찐고구마, 옥수수 추가 (간식 재료)
- 식빵 category `'곡류'` → `'빵 및 과자류'` 수정
- 우유 `serving_size` `'200ml'` → `'100ml'` 수정 (per-100ml 기준 정상화)

### page.note.today/api.py
- `_ai_analyze_all_meals()`: DB per-100g 결과에 `servings_per_food` 이중 곱셈 제거 → 순수 배식량 스케일만 적용
- `_classify_food_group()`: 죽류 분류 추가 (새우살죽 등 → 곡류), 찐 채소류(단호박/고구마/옥수수) 분류 추가
- 간식용 배식량 테이블 `_SNACK_SERVING_GRAMS` 추가 (곡류 40g, 채소류 80g 등)
- DB 결과 사용 시 끼니 유형(점심/간식)에 따라 배식량 자동 분기
