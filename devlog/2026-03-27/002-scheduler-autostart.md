# 알레르기 알람 08시 발송 수정

- **ID**: 002
- **날짜**: 2026-03-27
- **유형**: 버그 수정

## 작업 요약
알레르기 알림 스케줄러(`scripts/scheduler.py`)가 코드는 존재하나 프로세스가 자동 시작되지 않아 알림이 미발송되던 문제 수정. `run.sh`에 스케줄러 백그라운드 자동 시작 코드를 추가하고, 현재 서버에서 스케줄러를 즉시 시작.

## 변경 파일 목록

### run.sh
- `wiz run` 명령 전에 `nohup python3 scheduler.py &` 백그라운드 실행 추가
- `/var/log/wiz/scheduler.log`에 로그 기록
- 컨테이너/서버 재시작 시에도 자동 실행 보장

### 확인 사항
- `schedule` 패키지 설치 확인 (v1.2.2)
- KST 타임존 설정 정상 (`os.environ['TZ'] = 'Asia/Seoul'`)
- 스케줄러 시작 로그: `[2026-03-27 10:15:52] 스케줄러 시작 (KST) - 매일 08:00 알레르기 알림`
