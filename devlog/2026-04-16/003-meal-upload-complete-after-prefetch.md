# 업로드 완료 후 무지연 사용성 보장 검증

- **ID**: 003
- **날짜**: 2026-04-16
- **유형**: 버그 수정

## 작업 요약
식단표 업로드 프로그레스가 100%에 도달하는 시점을 월간 캘린더 데이터, 파일 목록, 월 전체 일간 상세 프리패치 완료 이후로 맞췄다.
이제 업로드 완료 직후에는 첫 날짜뿐 아니라 해당 월의 일간 상세가 미리 캐시에 준비되어, 사용자가 캘린더 진입/날짜 클릭/좌우 이동을 해도 추가 대기나 빈 화면이 발생할 가능성을 줄였다.

## 변경 파일 목록
- `src/app/page.note.meal/view.ts`
  - 업로드 완료 흐름에서 `prefetchCurrentMonthDailyData(true)`를 기다린 뒤 100% 완료 처리하도록 조정
  - 초기 진입, 뒤로가기, 월 변경, 재파싱, 삭제 후에도 월간 일간 프리패치/캐시 정리 흐름을 맞춤
- `src/app/page.note.meal/api.py`
  - 월간 응답 즉시 렌더링 보강을 위한 `photo_path` 유지

## 검증 메모
- 코드 흐름 기준으로 업로드 완료 전에 월간 데이터/파일 목록/일간 상세 캐시가 모두 준비되도록 정리함
- 정적 오류 검사는 완료했고, 실제 브라우저 상호작용은 이 환경에서 직접 실행하지 못함
