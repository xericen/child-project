# 식단표 업로드 완료 시점과 초기 진입 상태 동기화

- **ID**: 001
- **날짜**: 2026-04-16
- **유형**: 버그 수정

## 작업 요약
식단표 HWP 업로드 완료 직후와 일일 식단 화면 진입/이동 시 발생하던 빈 화면 문제를 수정했다.
월간 데이터로 일간 화면을 즉시 시드하고, 실제 일간 API 응답은 뒤에서 덮어쓰는 구조로 바꿔 `등록된 식단이 없습니다`가 잘못 노출되는 경로를 제거했다.

## 변경 파일 목록
- `src/app/page.note.meal/view.ts`
  - 일간 캐시(`dailyCache`)와 로딩 상태(`dailyLoading`) 추가
  - 월간 데이터 기반 즉시 렌더링(`seedDailyFromMonthly`) 추가
  - 업로드 완료 시 첫 식단 날짜를 미리 선택하고 일간 데이터까지 준비한 뒤 100% 완료 처리
  - 이전/다음 날짜 이동 시 빈 날짜가 아닌 실제 식단 날짜 기준으로 표시 유지
- `src/app/page.note.meal/view.pug`
  - 일간 식단 로딩 중 안내 문구 추가
  - 빈 상태 문구가 로딩 중 오표시되지 않도록 분기 수정
- `src/app/page.note.meal/api.py`
  - `get_monthly()` 응답에 `photo_path` 포함
  - 월간 응답만으로도 일간 화면의 기본 렌더링이 가능하도록 보강
